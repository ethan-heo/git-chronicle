## 1. 프로젝트 개요

### 1.1 배경 및 목적

이전 프로젝트에서 내가 작업한 내용을 분석하려면 Git 로그를 직접 탐색해야 한다. 이 과정이 비효율적이어서 프로그래매틱하게 자동화하는 VSCode Extension을 만든다.

### 1.2 핵심 기능 요약

| # | 기능 | 설명 |
|---|------|------|
| F-01 | 커밋 로그 조회 | 전체 커밋 이력을 목록으로 표시. 기간·작성자 필터, 커밋 메시지 키워드 검색, 내림차순 정렬 |
| F-02 | 변경 파일 트리 | 커밋 선택 시 변경 파일을 디렉토리 구조로 표시. AI 정리 저장본 존재 여부 뱃지 포함 |
| F-03 | 코드 변경이력 | 파일 단위 diff 뷰어 |
| F-04 | 의존 관계 캔버스 | 변경 파일 간 의존 관계를 노드-엣지 그래프로 시각화 |
| F-05 | AI 정리 (파일 단위) | 파일 변경 내역을 AI가 마크다운으로 요약. 로컬 자동 저장, 기존 저장본 재사용, 재생성 지원 |
| F-05b | AI 정리 (커밋 단위) | 커밋 전체 변경을 AI가 마크다운으로 종합 요약. 로컬 자동 저장 |
| F-08 | AI 정리 일괄 생성 | 커밋 내 모든 파일에 대해 파일 단위 AI 정리를 순차 자동 생성 |
| F-06 | AI 설정 | CLI AI(Claude / Gemini / Codex) 등록·활성화·비활성화 |
| F-07 | 저장 경로 설정 | AI 정리 결과물 저장 경로 지정 및 삭제 |

## 2. 화면 구성

확장 프로그램은 **퍼널(Funnel) 방식**으로 화면이 전환된다. 상위 화면에서 항목을 선택하면 세부 화면으로 진입하고, 뒤로가기 버튼으로 복귀한다.

| 화면 ID | 화면명 | 진입 조건 | 이전 화면 |
|---------|--------|-----------|-----------|
| S-01 | 커밋 목록 | 확장 프로그램 활성화 | — |
| S-02 | 이력 조회 | 커밋 목록에서 항목 클릭 | S-01 |
| S-03 | 코드 뷰어 | 파일/노드 호버 → 코드 버튼 클릭 | S-02 |
| S-04 | AI 정리 뷰어 | 파일/노드 호버 → AI 버튼 클릭, 또는 [커밋 AI 정리] 클릭 | S-02 |
| S-05 | 캔버스 | 이력 조회 화면 내 캔버스 버튼 클릭 | S-02 |
| S-06 | 설정 | 우측 상단 설정(⚙) 아이콘 클릭 | 어디서든 |

## 3. 사용자 시나리오

### 3.1 설정 (S-06)

1. 확장 프로그램 우측 상단의 설정(⚙) 아이콘을 클릭하면 설정 화면으로 전환된다.
2. **AI 등록** 영역에는 Claude / Gemini / Codex 각각의 버튼이 존재한다.
   - 비활성화 상태의 버튼을 클릭하면 CLI 연동 과정이 수행되고, 성공 시 버튼이 활성화 상태로 전환된다.
   - 활성화 상태의 버튼을 클릭하면 비활성화된다.
3. **저장 경로** 영역의 UI를 클릭하면 디렉토리 선택 다이얼로그가 활성화된다.
   - 경로 선택 후 확인을 누르면 해당 경로가 화면에 표시되고, 삭제 버튼이 활성화된다.
   - 삭제 버튼을 클릭하면 경로가 제거되고 초기 상태로 복원된다.
4. **[뒤로가기]** 버튼을 클릭하면 설정 진입 전 화면으로 복귀한다.

---

### 3.2 커밋 목록 (S-01)

1. 확장 프로그램 활성화 시 전체 커밋 이력이 목록 형태로 로드된다.
   - Git 저장소가 감지되지 않으면 "Git 저장소가 감지되지 않았습니다" 안내 메시지와 레포 열기 CTA가 표시된다.
2. 이력은 **커밋 생성 시간 기준 내림차순**으로 정렬된다.
3. 필터 영역에서 다음 기준으로 목록을 필터링할 수 있다.
   - **기간(날짜 범위)**
   - **작성자(Author)**
   - **커밋 메시지 키워드**: 텍스트 입력창에 키워드를 입력하면 커밋 메시지에 해당 단어가 포함된 항목만 표시된다.
4. 커밋 항목을 클릭하면 이력 조회 화면(S-02)으로 전환된다.
5. 필터 적용 결과 커밋이 없으면 "조건에 맞는 커밋이 없습니다" 안내 메시지를 표시한다.

---

### 3.3 이력 조회 (S-02)

1. 선택된 커밋의 변경 파일이 **디렉토리 트리** 형태로 표시된다.
2. 파일 항목에 마우스를 호버링하면 두 개의 액션 버튼이 활성화된다.
   - **[코드 보기]** → 코드 뷰어(S-03) 활성화
   - **[AI 정리 보기]** → AI 정리 뷰어(S-04) 활성화
     - 해당 파일의 AI 정리 저장본이 이미 존재하면 파일명 옆에 **"저장됨"** 뱃지가 표시되며, 클릭 시 기존 저장본을 즉시 불러온다.
     - 저장본이 없으면 AI를 호출하여 새로 생성한다.
3. 화면 상단에는 커밋 단위 액션 버튼이 위치한다.
   - **[커밋 AI 정리]** → 해당 커밋 전체 변경을 종합 요약한 AI 정리 뷰어(S-04) 활성화
   - **[전체 파일 AI 정리]** → 파일 트리의 모든 파일에 대해 파일 단위 AI 정리를 순차 생성. 화면을 이동해도 백그라운드에서 계속 진행되며 상단 고정 프로그레스 바로 진행 상태를 확인할 수 있다. [취소] 버튼으로 중단할 수 있으며, 완료 시 토스트 알림과 함께 각 파일에 "저장됨" 뱃지가 표시된다.
   - **[캔버스 보기]** → 의존 관계 캔버스(S-05) 활성화
4. **[뒤로가기]** 버튼을 클릭하면 커밋 목록(S-01)으로 복귀한다.

---

### 3.4 의존 관계 캔버스 (S-05)

1. 이력 조회 화면에서 [캔버스 보기] 버튼을 클릭하면 캔버스 화면이 활성화된다.
2. 변경 파일이 **노드(Node)** 로 표시되고, 파일 간 의존 관계가 **엣지(Edge)** 로 연결된다.
3. 노드에 마우스를 호버링하면 두 개의 액션 버튼이 활성화된다.
   - **[코드 보기]** → 코드 뷰어(S-03) 활성화
   - **[AI 정리 보기]** → AI 정리 뷰어(S-04) 활성화

---

### 3.5 AI 정리 뷰어 (S-04)

1. 파일 트리 또는 캔버스 노드에서 [AI 정리 보기] 버튼을 클릭하거나, [커밋 AI 정리] 버튼을 클릭하면 AI 정리 화면이 활성화된다. 화면 상단 헤더에는 **`{커밋 메시지} > {파일 경로}`** 형태로 현재 맥락이 표시된다. (커밋 단위 정리의 경우 `{커밋 메시지} > 커밋 전체 요약` 표시)
2. **기존 저장본이 있는 경우**: 저장된 마크다운 파일을 즉시 불러와 표시한다.
   - **[재생성]** 버튼으로 AI를 다시 호출하여 내용을 교체할 수 있다.
3. **저장본이 없는 경우**: 클릭 즉시 해당 파일(또는 커밋 전체)의 변경 내용을 컨텍스트로 AI가 마크다운 형식으로 정리한다.
   - 정리된 내용은 설정된 경로에 자동 저장된다.
   - 저장 경로가 설정되어 있지 않으면 "저장 경로를 먼저 설정해주세요" 안내 메시지와 설정 이동 CTA가 표시된다.
4. AI가 설정되어 있지 않은 상태에서 AI 정리를 시도하면 "AI가 설정되지 않았습니다" 안내 메시지와 설정(⚙) 이동 CTA가 표시된다.
5. 결과물이 마음에 들지 않을 경우 **[재생성]** 버튼으로 다시 정리를 요청할 수 있다.

---

### 3.6 코드 뷰어 (S-03)

- 파일 트리 또는 캔버스 노드에서 [코드 보기] 버튼을 클릭하면 해당 파일의 **변경 이력(diff)** 을 확인할 수 있다.

## 4. 기능 상세

### 4.1 AI 정리

#### 파일 단위 (F-05)

| 항목 | 내용 |
|------|------|
| 입력 | 파일 diff (변경 전/후 코드) |
| 처리 | 설정된 AI CLI에 기본 프롬프트 + diff 전달 (`child_process.spawn` 스트리밍) |
| 출력 | 마크다운 형식의 작업 내용 요약 (스트리밍 타이핑 효과로 실시간 표시) |
| 저장 | `{설정경로}/{폴더명}/{파일명}.md` 로컬 저장 |
| 기존 저장본 | 저장본이 존재하면 AI 호출 없이 파일을 즉시 표시 |
| 재생성 | [재생성] 버튼 클릭 → 덮어쓰기 확인 다이얼로그 → 확인 시 동일 입력으로 재처리 |
| 토큰 한계 초과 | "diff가 큽니다. AI가 일부를 생략할 수 있습니다" 안내 표시 후 그대로 호출 |
| 실패 / 타임아웃 | 타임아웃 120초. 실패 시 "생성에 실패했습니다" 오류 메시지 + [재시도] 버튼 표시 |
| 기본 프롬프트 | 아래 파일 단위 프롬프트 명세 참고 |

#### 커밋 단위 (F-05b)

| 항목 | 내용 |
|------|------|
| 입력 | 커밋 내 전체 파일 diff 합산 |

| 출력 | 마크다운 형식의 커밋 종합 요약 (스트리밍 타이핑 효과로 실시간 표시) |
| 저장 | `{설정경로}/{폴더명}/_commit_summary.md` 로컬 저장 |
| 기존 저장본 | 저장본이 존재하면 AI 호출 없이 파일을 즉시 표시 |
| 재생성 | [재생성] 버튼 클릭 → 덮어쓰기 확인 다이얼로그 → 확인 시 동일 입력으로 재처리 |
| 토큰 한계 초과 | "diff가 큽니다. AI가 일부를 생략할 수 있습니다" 안내 표시 후 그대로 호출 |
| 실패 / 타임아웃 | 타임아웃 120초. 실패 시 "생성에 실패했습니다" 오류 메시지 + [재시도] 버튼 표시 |
| 기본 프롬프트 | 아래 커밋 단위 프롬프트 명세 참고 |

#### AI 정리 일괄 생성 (F-08)

| 항목 | 내용 |
|------|------|
| 진입 | S-02(이력 조회)의 [전체 파일 AI 정리] 버튼 |
| 동작 | 변경 파일 트리의 모든 파일에 대해 파일 단위 AI 정리를 순차적으로 생성 |
| 중복 처리 | 이미 저장본이 존재하는 파일은 건너뜀 (스킵) |
| 에러 처리 | 개별 파일 실패 시 해당 파일을 건너뛰고 계속 진행. 완료 후 "실패 N개" 요약 토스트 표시 |
| 취소 | 상단 고정 프로그레스 바의 [취소] 버튼으로 중단 가능. 취소 시 완료된 파일의 저장본은 유지 |
| 진행 표시 | "n / 전체" 형태의 진행 상태 표시. 화면 이동 시에도 상단 고정 프로그레스 바로 계속 표시 |
| 완료 | 각 파일에 "저장됨" 뱃지 표시 + 완료 토스트 알림 |

### 4.2 AI 설정

| 항목 | 내용 |
|------|------|
| 지원 AI | Claude CLI, Gemini CLI, Codex CLI |
| 연동 방식 | CLI 인터페이스 기반 (`child_process.spawn`) |
| 연동 확인 | 등록 시 `{cli} --version` 실행 → exit code 0이면 성공, 비정상 종료면 실패 |
| CLI 미설치 | 등록 시도 후 실패 시 "CLI가 감지되지 않습니다. 설치 페이지를 확인하세요" 안내 + 설치 링크 표시 |
| CLI 사후 제거 | AI 정리 호출 시점에 실행 실패로 감지 → "연결된 CLI를 찾을 수 없습니다. 설정을 확인하세요" 안내 |
| 상태 관리 | 활성화 / 비활성화 토글 |
| 다중 등록 | 복수 등록 가능, 단 하나만 활성화 가능. 하나가 활성화되면 나머지는 자동으로 비활성화 |

### 4.3 의존 관계 캔버스

| 항목 | 내용 |
|------|------|
| 분석 도구 | dependency-cruiser (JS/TS/CJS/ESM, TypeScript path alias 지원) |
| 렌더링 라이브러리 | React Flow (MIT 라이선스, 줌·패닝·선택 인터랙션 내장) |
| 노드 범위 | 커밋에서 변경된 파일만 (의존하는 미변경 파일은 노드로 표시하지 않음) |
| 엣지 | 변경 파일 간 import / require 의존 관계 |
| 고립 노드 | 의존 관계가 없는 변경 파일도 노드로 표시 (엣지 없이 단독 노드) |
| JS/TS 외 파일 | 노드로 표시하되 엣지 없음. 노드에 "의존 관계 분석 불가" 툴팁 표시 |
| 레이아웃 | 자동 배치 (force-directed) |
| 인터랙션 | 노드 호버 → [코드 보기] / [AI 정리 보기] 버튼 활성화 |

### 4.4 저장 경로 설정

| 항목 | 내용 |
|------|------|
| 설정 방법 | 디렉토리 선택 다이얼로그 |
| 저장 구조 | `{루트경로}/{폴더명}/{파일명}.md`, 커밋 단위는 `_commit_summary.md` |
| 폴더명 규칙 | 커밋 메시지 앞 50자. 파일시스템 불가 특수문자(`/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `\|`)는 `_`로 치환 |
| 미설정 시 동작 | AI 정리 시도 시 "저장 경로를 먼저 설정해주세요" 토스트 + 설정(⚙) 화면 자동 진입 |
| 경로 자동 생성 | 지정된 경로가 존재하지 않으면 `fs.mkdirSync({ recursive: true })`로 자동 생성 |
| 삭제 | 삭제 버튼 클릭 시 경로 초기화 (기존 파일은 삭제하지 않음) |

### 4.6 커밋 로그 조회 (F-01)

| 항목 | 내용 |
|------|------|
| 로딩 전략 | git 명령어 레벨 필터 (`--max-count`, `--after`, `--before`, `--author`, `--grep`) + 무한 스크롤 |
| 초기 로드 | 200개. 스크롤 하단 도달 시 200개씩 추가 로드 |
| 날짜 필터 | `<input type="date">` × 2 (시작일 / 종료일) |
| 작성자 필터 | 드롭다운 자동완성 (커밋 로드 시 작성자 목록 추출하여 선택지 표시) |
| 키워드 검색 | 커밋 메시지 대상 (`--grep`). 입력 후 300ms 디바운싱 |
| 필터 조합 | 세 조건(기간·작성자·키워드) AND 고정 |
| 정렬 | 커밋 생성 시간 기준 내림차순 고정 |
